Description
===========

This sample illustrates a RESTful service that comprises of three resources and how you can build the business logic for each resource.

- ecommerceservice
    /products
    /orders
    /customers



How to run this sample
======================
bin$ ./ballerina run ../samples/restfulService/ecommerceService.balx

This starts the EcommerceService as well as the other backend services required to run the sample.



Invoking the service
====================


curl -X GET "http://localhost:9090/ecommerceservice/products/123000"

curl -X POST -d '{ "Product": { "ID": "123456", "Name": "XYZ", "Description": "Sample product."}}'  "http://localhost:9090/ecommerceservice/products" -H "Content-Type:application/json"



curl -X GET "http://localhost:9090/ecommerceservice/orders"

curl -X POST -d '{ "Order": { "ID": "111999", "Name": "XYZ", "Description": "Sample order."}}'  "http://localhost:9090/ecommerceservice/orders" -H "Content-Type:application/json"



curl -X GET "http://localhost:9090/ecommerceservice/customers"

curl -X POST -d '{"Customer": {"ID": "987654", "Name": "ABC PQR","Description": "Sample Customer."}}'  "http://localhost:9090/ecommerceservice/customers" -H "Content-Type:application/json"
